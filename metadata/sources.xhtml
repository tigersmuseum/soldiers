Sources

We create a list of Soldiers XML person entries from some source, and we give that list a source label (see below). If we can successfully identify a person entry from such a list, then we have a database Soldier ID (SID) associated with a source. Doing this for all available sources builds a map of SID to Source. This map lets us find all the source entries that relate to a specific soldier. The source-specific notes in these entries can be collated to pull together biographical data.

There are 3 attributes on the Soldiers XML note element that relate to source:

sourceref
A formal source reference or identifier that should be used when the source is a particular historical document.
source
A name for the source.
type
The type of the source material. This offers a way of grouping similar records from different resources together. For example, "medals" or "medical".

Ideally, we would like to capture enough information in a Soldiers XML note element to allow it to be traced back to a specific historical document. The sourceref attribute allows this provided that it is available and can be sensibly de-referenced. The source attribute should give the context to uniquely identify the referenced document.

It may be that links to historical records are lost because the source to the creator of the Soldiers XML data is some intermediary that hasn't kept a detailed record of its own sources. The intermediary might be able to generate its own reference if it maintains a database of the records it generates, but this isn't to be expected. Instead, we will have a named source for a note, but without a source reference.
